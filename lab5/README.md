## 核心概念

### 為什麼要評估？

訓練時看到 reward 上升不代表模型真的變好了，我們需要用**獨立的測試案例**來驗證：

```
訓練資料 → 模型學習的依據（不應拿來評估）
測試資料 → 獨立的案例，用來驗證模型是否真正學會
```

### 評估指標

| 指標 | 說明 |
|------|------|
| 平均 Reward | 使用與訓練相同的 reward function 計算 |
| JSON 正確率 | 模型輸出是否為有效的 JSON 格式 |
| 工具正確率 | 是否選擇了正確的工具 |

### Step 2：觀察輸出

程式會依序：
1. 載入基礎模型（訓練前）
2. 對測試案例進行推論並計算分數
3. 載入訓練後的模型（LoRA adapter）
4. 對相同測試案例進行推論並計算分數
5. 比較兩者的表現差異

## 預期結果

評估完成後，你應該看到：

1. ✅ 訓練後模型的 `reward/mean` 高於訓練前
2. ✅ 模型輸出更符合 JSON 格式
3. ✅ 工具選擇和參數填寫更準確
